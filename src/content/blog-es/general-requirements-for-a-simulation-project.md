Cuando potenciales clientes (ya sea un estudiante o una gran organización) nos contactan para hacer un modelo de simulación, a menudo preguntan, antes de enviar cualquier requisito, qué necesita tener este requisito, para ser lo suficientemente completo para evaluar costos, factibilidad y cronograma. En 2018, escribimos un artículo indicando cómo reducir la ambigüedad de tu proyecto para hacerlo más objetivo, con el fin de maximizar el valor con el menor precio posible, puedes leerlo aquí: [/es/blog/how-to-optimize-the-cost-and-quality-of-your-project-as-a-client/](/es/blog/how-to-optimize-the-cost-and-quality-of-your-project-as-a-client/)

Pero este artículo está destinado a clientes, estudiantes o cualquier persona interesada en contratar a alguien para construir un modelo de simulación. ¿Qué debería tener tu documento de requisitos? Esto es sobre lo que escribiremos en este artículo.

## Propósito

El propósito es lo más importante que necesitas definir. Esto identificará las características generales que necesitan estar presentes, como animación 3D, interfaz hermosa, integración con otras plataformas, nivel de detalle, importancia de dashboards, etc.

En general quieres responder las siguientes preguntas:

- ¿Por qué necesitas construir un modelo de simulación? ¿Qué problema estás tratando de resolver? ¿Cuál es tu pregunta de investigación?

¿Qué resultado esperas cuando el modelo de simulación esté listo?

- ¿Un diseño de una instalación?

- ¿Una política o estrategia identificada?

- ¿Una entidad de inteligencia artificial?

- ¿Una herramienta para uso a largo plazo o para una decisión estratégica única?

¿Quién es la audiencia de esta simulación?

- ¿El modelo se usa para recaudar fondos, o construir confianza con socios o directivos?

- ¿El modelo es usado por operadores que aplicarán resultados de manera regular?

- ¿El modelo se usa para generar datos que serán útiles para probar/refutar un punto?

- ¿El modelo se usa para optimizar algo? ¿Programación? ¿Toma de decisiones automatizada? ¿Algo más?

¿Quiénes son las partes interesadas de este proyecto, usuarios de la herramienta de simulación y personas interesadas en sus resultados?

- ¿Directores? ¿Inversionistas? ¿Operadores? ¿Personal administrativo? ¿Científicos de datos? ¿Alguien más?

- ¿Por qué a cada uno de estos individuos les importa este trabajo?

- ¿Por qué crees que este trabajo es importante?

## El Sistema

Ya sea que estés construyendo una instalación de manufactura que tiene procesos claros (o no claros), una estrategia de marketing que entiende a los individuos, o una estrategia gubernamental de alto nivel que entiende el panorama político, necesitamos ser claros sobre la complejidad del sistema para identificar los esfuerzos requeridos para hacerlo, y la tecnología que necesita usarse.
Así que necesitamos saber:

- Si hay algún layout del sistema o archivo shape del área, o cualquier representación física del sistema, envíalo con la explicación necesaria que nos permita entenderlo.

- Necesitamos entender el sistema al menos a un alto nivel. Para una planta de manufactura o un almacén, esto será el flujo del proceso, si lo tienes, o una descripción de alto nivel de este proceso si no lo tienes. Si es algo diferente, cualquier diagrama, junto con la explicación de ese diagrama al nivel que puedas proporcionar. Si no tienes idea sobre el sistema, entonces déjanos claro qué tú (o nosotros) necesitamos investigar, en cuyo caso necesitamos pasar por una fase exploratoria y cotizaríamos por eso.

¿Cuál es la tecnología utilizada en el sistema? Sé lo más explícito posible con la información que conoces.

- Equipos de manejo de materiales

- Recursos humanos

- Robótica

- Cualquier otro actor del sistema en el que necesitemos pensar

## Resultados y salidas

Los resultados y salidas del sistema son parte del propósito de este proyecto y se utilizan para poder contar la historia después de que la simulación esté hecha. Lo que necesitamos saber aquí es:

- ¿Cuáles son los Indicadores Clave de Rendimiento (KPIs) más importantes que quieres obtener?

- ¿Qué tipo de simulación es esta? ¿Quieres investigar el sistema actual, o quieres comparar el sistema actual con un sistema futuro con diferentes escenarios? Si es este el caso, ¿cuáles son los escenarios que quieres investigar?

¿Quién va a ser la audiencia para:

- validar que el modelo es realmente una buena representación de la realidad.

- visualizar los resultados del modelo, y ¿cómo crees que sería mejor visualizarlos? Tenemos un video sobre lo que puedes obtener: [/es/simulation-services/](/es/simulation-services/). Es posible hacer algo extremadamente profesional, o algo rápido y básico, dependiendo de quién va a usar o mirar esto.

## Datos

Los datos no son necesarios para comenzar con una simulación, y podemos trabajar en los datos después ya que una simulación realmente no necesita datos siempre y cuando podamos usar supuestos. Los datos no son importantes para hacer una cotización. Sin embargo, esto tiene una excepción. Esta excepción es: ¿necesitamos limpiar tus datos? ¿Necesitamos trabajar con tus datos para poder usarlos? Si ese es el caso, danos toda la información necesaria para evaluar la cantidad de trabajo requerido para hacer esto.

## Conclusión

Esta es una idea genérica sobre lo que se usa para comenzar a hacer trabajo de simulación. En general, la cotización se basará en la ambigüedad del proyecto. Un proyecto con poca información se considera de alto riesgo, y los proyectos de alto riesgo tienden a ser más caros y tardan más. Un proyecto con toneladas de información detallada y de buena calidad tiende a ser más barato y se desarrolla más rápido.
