## Course Overview

This session demonstrates best practices for exporting data from simulation models. Participants will learn how to define parameters using custom classes, leverage the Lombok library for cleaner code, and implement professional export workflows to text files, Excel, and databases.

---

## Objectives

- Learn best practices for parameter definition using custom classes
- Understand the advantages of the Lombok library in AnyLogic and how to use it
- Learn to collect and save data systematically within a model
- Master data export techniques for simulation models in AnyLogic

---

## Prerequisites

**AnyLogic**
- Interface familiarity, View Areas, Agents, Events, Functions, Databases

**JAVA**
- Variables, Parameters, Casting

**Discrete Events**
- Source, Resources, Service, Seize, Release, MoveTo, Transporters, Delay, Sink

**Statistical Foundations**
- Uniform distribution, Triangular distribution

---

## Topics Covered

### 1. Parameter Definition

- Adding parameters using a custom Java class
- Reading parameter values from Excel at runtime

### 2. The Lombok Library

- What is the Lombok library and its advantages
- How to configure and use Lombok in AnyLogic projects

### 3. Exporting Data from Simulation Experiments

- Adding a class for each export table
- Saving class instances in collections — the export general class
- Exporting to text files
- Exporting to Excel files
- Exporting to a database

---

## Acquired Knowledge

By the end of this session, participants will be able to:

- Define parameters in a custom class and read values from Excel
- Configure the Lombok library in AnyLogic projects
- Implement structured data collection within simulation models
- Export simulation data to text files, Excel, or databases using professional patterns
